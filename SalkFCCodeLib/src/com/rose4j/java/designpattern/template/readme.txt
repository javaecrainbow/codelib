模板方法：
使用背景：
模板方法模式是通过把不变的的行为搬移到超类，
去除子类中重复的代码来体现它的优势；
当不变的和可变的行为在子类实现中混合在一起的时候，
不变的行为就会在子类中重复实现，
我们通过模板方法模式把这些行为搬移到单一的地方，
这样就可以帮助子类摆脱重复不变行为的纠缠。
认识模板方法：
模板方法定义了一个算法的步骤，并允许子类为一个或多个步骤提供实现
模板方法模式在一个方法中定义一个算法的骨架，而将一些步骤延迟到子类中，模板方法可以使得子类在不改变算法结构的情况下，重新定义算法中的某些步骤。
模板方法提高代码的复用，
模板方法中的算法一般是final，子类无法覆写，但是如果在能够更好的提高扩展性，我们常常在抽象类中放一个钩子，这个钩子的作用就是让子类去决定是否需要改变算法 
依赖倒置原则教我们尽量避免使用具体类，而使用抽象，好莱坞原则则是用在创建框架或组建上的一种技巧，好让低层组件能够被挂钩进计算中，而不让高层组件依赖于低层组建，两个原则都是实现解耦    
模板和策略模式的区别，一个用组合，一个用继承，策略中所组合的类实现了整个算法，但是在模板中子类是实现了算法中的其中一部分
